## Project Setup

This is a **SwiftUI iOS** Sudoku app. The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen).

```bash
# Install XcodeGen (once)
brew install xcodegen

# Generate the .xcodeproj
xcodegen generate

# Open in Xcode
open MobileSudoku.xcodeproj
```

Build and run via Xcode (⌘R) or `xcodebuild`:

```bash
# Build for simulator
xcodebuild -scheme MobileSudoku -destination 'platform=iOS Simulator,name=iPhone 16' build

# Run tests (none yet — add XCTest targets to project.yml when needed)
xcodebuild test -scheme MobileSudoku -destination 'platform=iOS Simulator,name=iPhone 16'
```

## Architecture

**MVVM** with pure-value models. No third-party dependencies.

```
MobileSudoku/
├── App/            Entry point (MobileSudokuApp.swift)
├── Models/         Pure-value structs + algorithms
├── ViewModels/     ObservableObject classes
└── Views/          SwiftUI views
```

### Data flow

```
SudokuGenerator → SudokuBoard (struct) ← GameViewModel (@MainActor ObservableObject)
SudokuSolver ──────────────────────────────────────↑ (used at generation & hint time)
```

`GameViewModel` owns the `SudokuBoard` and drives all view updates. Views read from the VM via `@StateObject`/`@ObservedObject` and call VM methods for user actions.

### Key files

| File | Role |
|---|---|
| `Models/SudokuSolver.swift` | Backtracking solver; also used to verify unique solutions |
| `Models/SudokuGenerator.swift` | Builds a full solution, then digs holes while preserving uniqueness |
| `Models/SudokuBoard.swift` | Board state, error validation, hint provider |
| `Models/CellState.swift` | Per-cell data: value, isGiven, candidates (pencil marks), isError |
| `Models/Difficulty.swift` | Difficulty enum with clue counts, hint limits, score bases |
| `ViewModels/GameViewModel.swift` | All game logic: selection, input, hints, timer, scoring |
| `Views/GameView.swift` | Main game screen; hosts header, BoardView, control bar, NumberPadView |
| `Views/BoardView.swift` | 9×9 cell grid + Canvas-drawn grid lines |
| `Views/CellView.swift` | Individual cell with pencil-mark sub-grid (CandidatesView) |

### Algorithms

**Puzzle generation** (`SudokuGenerator`): randomized backtracking fills the grid, then cells are removed one at a time — a cell is only removed if the resulting board still has a unique solution (verified by `SudokuSolver.hasUniqueSolution`). Harder difficulties remove more cells.

**Solver** (`SudokuSolver`): standard backtracking. `hasUniqueSolution` counts solutions up to 2 and short-circuits; `solution(for:)` returns the first found solution.

**Error highlighting** (`SudokuBoard.validateErrors`): after every value change, scans all rows, columns, and 3×3 boxes and sets `isError` on any duplicates.

**Candidate/pencil marks**: stored as `Set<Int>` in `CellState.candidates`; toggled individually via `GameViewModel.inputNumber` when `inputMode == .candidate`. Setting a definitive value clears candidates automatically.

### Input modes

`GameViewModel.inputMode` is either `.normal` (number tap sets the cell value) or `.candidate` (number tap toggles a pencil mark). Toggled by the 候选 button in `GameView.controlBar`.

### Scoring

`score = max(0, difficulty.scoreBase - timePenalty - mistakePenalty)`
where `timePenalty = min(elapsedSeconds * 2, scoreBase / 2)` and `mistakePenalty = mistakeCount * 50`.

## Configuration

`project.yml` is the XcodeGen source of truth for the Xcode project. Edit it (not the `.xcodeproj`) when adding new targets, build settings, or file groups, then re-run `xcodegen generate`.

Minimum deployment target: **iOS 17.0**. Uses `onChange(of:)` two-argument form and `Color(.systemGray5)` — do not lower the target without auditing for API availability.
